## Feedback-Details

### F-01: CD/Cooldown-Terminologie entfernen

- **Priorität:** P0
- **Bereich:** UI / Copy
- **Typ:** Fix (Sweep)
- **Status:** ✅ erledigt (Batch 1)

**Beobachtung:** An diversen Stellen steht noch „CD“ bzw. Cooldown — das System hat keine Cooldowns mehr.

**Erwartung:** Keine Cooldown-Referenzen mehr in UI, Tooltips, How to Play, Kampflog o.ä.

**Hinweise:** Codebase-Sweep über `src/`, UI-Strings, ggf. veraltete Docs ignorieren (nur spieler-sichtbar).

**Umsetzung:** Einzige spieler-sichtbare Stelle war die `CD ${spell.cooldown}`-Anzeige in der Zauberleiste (`renderBuildCard`, `src/renderer.js`) — da alle Zauber `cooldown: 0` haben, zeigte das immer „CD 0“. Span + zugehörige `.spell-cooldown`-CSS-Klasse entfernt. Interne `cooldown`/`ignoreCooldown`-Felder in `spellEngine.js`, `combatContext.js`, `combatPrep.js` bewusst unangetastet gelassen (kein Spieler-Text, reine interne Datenfelder — Entfernen des Cooldown-Systems selbst wäre ein größerer Refactor, kein Quick Fix).

### F-02: How to Play — Statuseffekte aktualisieren

- **Priorität:** P0
- **Bereich:** Docs / UI
- **Typ:** Fix
- **Status:** ✅ erledigt (Batch 1)

**Beobachtung:** Im „How to Play“ sind noch alte Statuseffekte dokumentiert.

**Erwartung:** Aktuelle Statuseffekte des Spiels — konsistent mit Combat-System und Tooltip-Terminologie.

**Follow-up für Batch 6:** Das `KEYWORD_TOOLTIPS`-Glossar in `src/spellRegistry.js` (Zeilen 3–25) enthält dieselben veralteten Begriffe (Momentum, Timing, Traum, Urgewalt, Meisterrune, Blutrausch, Blutpakt, Runenverbindung, …). Aktuell wirkungslos (kein Zauber-Text matcht diese Wörter), aber sollte im globalen Tooltip-Pass bereinigt werden.

### F-03: Starter-Auswahl — Hover-Effekt fehlt

- **Priorität:** P1
- **Bereich:** UI
- **Typ:** Fix (Regression)
- **Status:** ✅ erledigt (Batch 1) — **bitte im Spiel gegenprüfen**

**Beobachtung:** Hover-Effekt auf Starter-Auswahl-Karten fehlt (war früher da).

**Erwartung:** Sichtbarer Hover-State wie zuvor (Highlight, Scale, Border o.ä.).

**Umsetzung:** Der Hover existierte technisch bereits (`transform`/`box-shadow` in `style.css`), war aber kaum wahrnehmbar: kein `transition` (Effekt „sprang“ ohne Animation) und die Rahmen-Verdunklung änderte sich nur minimal (Brightness 0.92 → 0.98). Anders als Belohnungskarten gibt es für Starterkarten kein eigenes Hover-Frame-Asset (nur `starter_card.png`, kein `starter_card_hover.png`). Verstärkt: spürbare Transition, deutlichere Aufhellung (0.92 → 1.18) und ein zusätzlicher Glow-Ring beim Hover, klar unterscheidbar vom „selected“-Zustand.

**Offener Punkt:** Falls gewünscht, könnte für Batch 5 (Starter-Balance) oder separat ein eigenes `starter_card_hover.png`-Asset erzeugt werden, analog zu den Reward-Karten.

### F-11: Belohnungs-Tooltip — LoL-Style Stat-Delta

- **Priorität:** P1
- **Bereich:** Reward UI / Tooltips
- **Typ:** UX
- **Status:** ✅ erledigt

**Beobachtung:** Verbesserung durch Upgrade nicht konkret genug sichtbar.

**Erwartung:** Explizite Vorher/Nachher-Anzeige, analog League-of-Legends-Rangerhöhung:
- Beispiel: `25 Schaden ➔ 35 Schaden`
- Nur die sich ändernden Werte hervorheben

**Umsetzung:** Es gab bereits eine vollständige, aber nie verdrahtete Funktion (`getUpgradeChangeLines` in `src/spellRegistry.js`), die genau dieses Vorher/Nachher-Format erzeugt (`30 → 40 Schaden`, `NEU: ...` für neue Mechaniken) über ~40 verschiedene Zauberwerte hinweg. Diese ist jetzt an `getSpellRewardView` angebunden und ersetzt die bisherige reine Anzeige des Ziel-Rang-Textes. Fallback auf die alte Tooltip-Vorschau, falls für einen Upgrade-Schritt keine erkennbaren Wertänderungen vorliegen. Pfadwahl-Karten zeigen ebenfalls Deltas statt reinem Fließtext, wo verfügbar.

### F-13: Belohnungs-Randomisierung & flexible Slot-Typen

- **Priorität:** P2
- **Bereich:** Reward System
- **Typ:** Design + Implementierung
- **Status:** ✅ erledigt

**Beobachtung:** Zu wenig Random; Chance auf passende Zauber zu gering. Alle 3 Felder scheinen fest vorzuschreiben, ob Upgrade oder Neuer Zauber.

**Erwartung:** Neues System entwickeln:
- Kein fester Slot-Typ (Upgrade/Neu) pro Feld
- Mehr Varianz, aber weiterhin Möglichkeit, Builds zusammenzubekommen
- Passende Zauber häufiger, ohne Random komplett zu eliminieren

**Umsetzung:** Unabhängige Slot-Würfe (55/45) mit Soft-Guarantee (mind. 1 Upgrade und 1 Neu, wenn beide Pools verfügbar). Schule-Affinität 1.5× für Zauber gleicher Schule wie im Build. Details in `docs/design/Reward_System_v2.md`.

### F-14: Drop-System — Zauberrang bei neuen Zaubern

- **Priorität:** P2
- **Bereich:** Reward System
- **Typ:** Design + Implementierung
- **Status:** ✅ erledigt

**Beobachtung:** Neue Zauber sollten nicht sofort hohe Ränge haben; Progression fehlt.

**Erwartung:**
- Neue Zauber standardmäßig **Rang 1**
- Mit Run-Fortschritt steigende Chance auf höhere Start-Ränge
- Konkretes Drop-Table + Steigerungskurve definieren und implementieren

**Umsetzung:** Start-Rang max. **2** (kein Pfad-Drop). Kurve: fight 0–2 → 95/5, 3–6 → 80/20, 7+ → 60/40. Option trägt `startRank`; `initializeSpellProgress(id, startRank)` + UI-Badge „Neuer Zauber · Rang II“. Spec: `docs/design/Reward_System_v2.md`.

**Abhängigkeit:** Mit F-13 gemeinsam als „Reward System v2“.

### F-15: Starter-Seltenheit & Powerlevel ausbalancieren

- **Priorität:** P2
- **Bereich:** Balance / Starter
- **Typ:** Balance + Content
- **Status:** ✅ erledigt

**Beobachtung:** Build-Enabler in Starterauswahl haben zu unterschiedliches Powerlevel; epische Spells tauchen auf; zu wenig „basic“-Spells.

**Erwartung:**
- Deutlich mehr Basic/Common-Spells in der Auswahl
- Seltenheit korrekt gewichtet (keine Epics in Starter-Pool, oder stark reduziert)
- Mehr Abwechslung bei vergleichbarem Powerlevel

**Umsetzung:**
- Epics (`death_stroke`, `shadow_dance`) aus dem Starter-Pool entfernt
- 5 Commons als Starter ergänzt: Blutgerinnsel, Knochenpanzer, Finsterer Hieb, Gedankenschlag, Gedankenbarriere
- Offer-Generierung gewichtet (Common 70 / Rare 28 / Epic 0) + Soft-Guarantee mind. 2 Commons pro 5er-Offer
- Enabler angeglichen: Präzisionsschlag 35/+30, Arkane Verkettung & Reinheit 30/+30

### F-16: Globaler Tooltip-Overhaul

- **Priorität:** P3
- **Bereich:** Content / Tooltips
- **Typ:** Content-Pass (groß)
- **Status:** ✅ erledigt

**Beobachtung:** Viele Tooltips enthalten interne Beschreibungen statt spielerfreundlicher Texte.

**Erwartung — einheitliches Tooltip-Schema:**

1. **Fantasy:** 1 kurzer Satz zur Zauberfantasy
2. **Effekt:** Kompakt & konkret, mit Zahlen
3. **Rang:** Aktueller Zauberrang *(außer Startauswahl)*
4. **Upgrades:** Weiterentwickelte Effekte durch **Rang 3** und **Rang 5** — additiv, überschreiben Basis-Wirkung nicht

**Umsetzung:**
- Tooltip-UI zeigt Fantasy → Effekt → Rang → Spezialisierung → „Weitere Entwicklung“ (Rang III/V-Vorschau, inkl. beide Pfade vor der Wahl)
- Startauswahl: kein Rang, Upgrade-Vorschau bleibt sichtbar
- Alle 35 Spell-`description`-Texte von Meta/Build-Sprache auf Fantasy umgeschrieben
- `KEYWORD_TOOLTIPS` auf aktuelle Mechaniken reduziert (Verwundbar, Schild, Krit, Vorbereitung)
- Effektzeilen bei unvollständigen Rang-Tooltips additiv gemerged (Basis bleibt erhalten)

**Hinweis:** Reward-Tooltips (F-10/F-11) bleiben separat; globales Hover-Schema gilt jetzt überall.
