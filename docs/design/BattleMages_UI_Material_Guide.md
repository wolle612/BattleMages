# BattleMages UI Material Guide

Version: 1.0

Dieses Dokument definiert die Materialsprache aller UI-Assets von
BattleMages. Zusammen mit `BattleMages_Assets_Master_Style_Prompt.md`
und `BattleMages_UI_Asset_List.md` bildet es die verbindliche Grundlage
für sämtliche UI-Komponenten.

------------------------------------------------------------------------

# Leitprinzip

Die Benutzeroberfläche wirkt wie ein uraltes Artefakt aus einer
verbotenen Magierakademie.

Nicht luxuriös.

Nicht verfallen.

Nicht überladen.

Sondern massiv, würdevoll und zeitlos.

Lesbarkeit steht immer über Detailgrad.

------------------------------------------------------------------------

# Licht

-   Primäre Lichtquelle: oben links
-   Sanfte Kantenlichter
-   Tiefe, kühle Schatten
-   Keine harten Glanzpunkte
-   Keine Spiegelungen

------------------------------------------------------------------------

# Basalt

**Verwendung**

-   Panels
-   Karten
-   Rahmen
-   Leisten
-   Altäre

**Eigenschaften**

-   Dunkles Anthrazit
-   Kalte Grautöne
-   Feine natürliche Steinstruktur
-   Kleine Risse
-   Leicht verwitterte Kanten
-   Keine groben Brüche
-   Ruhige Flächen

Basalt ist immer das dominante Material.

------------------------------------------------------------------------

# Bronze

**Verwendung**

-   Rahmen
-   Beschläge
-   Zierleisten
-   Einfassungen

**Eigenschaften**

-   Dunkle gealterte Bronze
-   Wenig Sättigung
-   Matt
-   Leichte Abnutzung
-   Dezente Kratzer
-   Keine Hochglanzoptik
-   Keine goldene Luxuswirkung

Bronze unterstützt den Stein -- sie dominiert ihn nicht.

------------------------------------------------------------------------

# Silber

**Verwendung**

-   Schild-Elemente
-   Magische Schutzobjekte

**Eigenschaften**

-   Kaltes Metall
-   Matt
-   Dezente Runengravuren
-   Leichte kristalline Anmutung
-   Keine Spiegelungen

------------------------------------------------------------------------

# Runengravuren

**Position**

-   Ecken
-   Beschläge
-   Abschlussplatten

**Regeln**

-   Eingraviert
-   Klein
-   Klar lesbar
-   Symmetrisch
-   Nie auf großen Innenflächen
-   Keine übermäßige Verzierung

Standardzustand: nicht leuchtend.

------------------------------------------------------------------------

# Pergament

Nur für Tooltip-Innenflächen.

-   Dunkel
-   Leicht vergilbt
-   Ruhig
-   Textfreundlich
-   Keine starke Textur

------------------------------------------------------------------------

# Kristalle

Nur sehr sparsam einsetzen.

-   Klein
-   Matt
-   Gedämpfte Leuchtkraft
-   Kein Fokusobjekt

------------------------------------------------------------------------

# Alterung

Alle Materialien besitzen:

-   leichte Kratzer
-   dezente Kantenabnutzung
-   feine Verwitterung

Keine:

-   Rostlöcher
-   Schmutzschichten
-   Moos
-   Blut
-   Ruß

------------------------------------------------------------------------

# Pixel-Art-Regeln

-   Production-Quality Pixel Art
-   Mittlerer Detailgrad
-   Klare Formen
-   Starke Silhouetten
-   Sichtbare Pixelstruktur
-   Handgesetzte Pixelcluster
-   Scharfe Pixelkanten
-   Keine heruntergerechnete Concept Art
-   Keine glatten Farbverläufe

Jedes Detail muss auch in kleiner Darstellung einen Zweck erfüllen.

------------------------------------------------------------------------

# Material-Hierarchie

1.  Basalt
2.  Bronze oder Silber
3.  Runengravuren
4.  Magische Akzente

Magische Effekte dürfen niemals das Material überdecken.

------------------------------------------------------------------------

# Verboten

-   Hochglanz
-   Edelsteinüberladung
-   Elfenornamente
-   Schädel
-   Dämonische Verzierungen
-   Gotische Spitzen
-   Neon
-   Sci-Fi
-   Comicstil
-   Goldluxus
-   Übermäßige Mikrodetails

------------------------------------------------------------------------

# Qualitätscheck

Vor Freigabe jedes Assets prüfen:

-   Wirkt der Stein schwer?
-   Ist Bronze klar von Stein getrennt?
-   Sind Runen dezent?
-   Bleibt die Innenfläche ruhig?
-   Ist das Material bei Spielgröße lesbar?
-   Passt das Asset zu allen bereits erstellten UI-Elementen?

Nur wenn alle Fragen mit Ja beantwortet werden, gilt ein Asset als
produktionsreif.
